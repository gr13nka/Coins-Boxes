Module structure (local M = {} ... return M):

**Constants:**
- SLIDE_DURATION = 0.3 (D-04)
- OVERLAY_OPACITY = 0.60 (D-02)
- PULSE_SPEED = 1.5 (cycles per second, D-03)
- HAND_TAP_PERIOD = 1.5 (seconds per tap cycle, D-14)
- HAND_DRAG_PERIOD = 2.0 (seconds per drag cycle, D-14)
- CUTOUT_PADDING = 12 (pixels around target element)
- CUTOUT_RADIUS = 12 (corner radius for rounded rect)
- TEXT_MARGIN = 20 (margin from spotlight edge)

**State variables (module-local):**
- active_tutorial = nil (nil, "coin_sort", or "arena")
- current_step = 0
- prev_rect = nil ({x, y, w, h})
- next_rect = nil ({x, y, w, h})
- slide_timer = 0
- pulse_timer = 0
- hand_timer = 0
- lang = "en" (default language, "en" or "ru")
- queued_advance = false (for animation-idle gating)

**Step definition tables:**
Create empty tables that Plan 02 and Plan 03 will populate:
```lua
local STEPS = {
    coin_sort = {},  -- populated by M.registerSteps("coin_sort", steps_table)
    arena = {},      -- populated by M.registerSteps("arena", steps_table)
}
```

Each step is a table with fields:
- text: {en = "...", ru = "..."} (D-16)
- getRect: function() returning {x, y, w, h} or nil (spotlight target in virtual canvas coords)
- hand: "tap" or "drag" or "none" (D-13, D-14)
- drag_target: function() returning {x, y} for drag destination (only when hand == "drag")
- check: function() returning true if precondition is met (D-17, TUT-04)
- on_enter: function() or nil, called when step becomes active (for setup actions like seeding dispenser)

**Public API functions:**

`M.registerSteps(tutorial_id, steps_table)` -- called by screen modules to register their step definitions. Stores into STEPS[tutorial_id].

`M.start(tutorial_id)` -- Sets active_tutorial, current_step = 1, calls step's on_enter if defined, sets next_rect from step's getRect(). Loads lang from a simple check (try Yandex SDK locale or default "en"; for now default to "ru" since Yandex is primary target, per D-16).

`M.update(dt)` -- Updates slide_timer (clamp at SLIDE_DURATION), pulse_timer (continuous), hand_timer (wraps at period). If queued_advance is true and require("animation").isIdle() returns true, calls internal doAdvance(). Returns nothing.

`M.advance()` -- Called by screen modules when the required action for the current step is completed. If animation is not idle (require("animation").isIdle() returns false), sets queued_advance = true and returns. Otherwise calls internal doAdvance() immediately.

Internal `doAdvance()` -- Increments current_step. If new step index > #STEPS[active_tutorial], calls M.markDone() and returns. Otherwise: validates new step's check() precondition. If check returns false, try up to 3 more steps forward looking for a valid one (skip broken steps gracefully per D-17). If no valid step found, markDone(). If valid: sets prev_rect = current interpolated rect, sets next_rect from new step's getRect(), resets slide_timer = 0, hand_timer = 0, queued_advance = false, calls step's on_enter if defined.

`M.isActive()` -- Returns active_tutorial ~= nil.

`M.getActiveTutorial()` -- Returns active_tutorial string or nil.

`M.getSpotlight()` -- Returns interpolated rect {x, y, w, h} using easeOutCubic between prev_rect and next_rect. If no prev_rect, returns next_rect directly. Returns nil if no active tutorial. Include CUTOUT_PADDING in the returned rect (expand by padding on all sides).

`M.getOverlayOpacity()` -- Returns OVERLAY_OPACITY (so screens don't hardcode it).

`M.getPulseAlpha()` -- Returns 0.4 + 0.4 * math.sin(pulse_timer * math.pi * 2 * PULSE_SPEED) (D-03 pulsing border brightness).

`M.getCutoutRadius()` -- Returns CUTOUT_RADIUS.

`M.getText()` -- Returns the localized text string for current step: STEPS[active_tutorial][current_step].text[lang]. Returns "" if no step or no text.

`M.getTextPosition(spotlight_rect)` -- Takes the spotlight rect. Returns "above" or "below" depending on which side has more space (D-15). If spotlight center_y < VH/2, return "below"; else return "above". Uses layout.VH for the comparison.

`M.getHandAnim()` -- Returns a table {type = "tap"/"drag"/"none", progress = 0-1, drag_target = {x,y} or nil}. Progress is hand_timer / period (wrapping). For "tap", progress 0-0.3 is hand moving down, 0.3-0.5 is hold, 0.5-1.0 is hand moving back up (tapping motion). For "drag", progress 0-0.8 is hand moving from spotlight center to drag_target, 0.8-1.0 is reset. The step's drag_target function provides the destination point.

`M.isInputAllowed(gx, gy)` -- Returns true if point (gx, gy) is inside the current spotlight rect (using the interpolated rect from getSpotlight()). Returns true if no active tutorial (pass-through when tutorial inactive). This is the inverse of the RESEARCH.md isInputBlocked pattern -- named "allowed" for clearer semantics.

`M.isDone(tutorial_id)` -- Checks progression data. Returns tutorial_data[tutorial_id .. "_done"] == true.

`M.markDone()` -- Sets active_tutorial to nil, current_step to 0, clears rects. Persists: loads tutorial_data from progression, sets [active_tutorial .. "_done"] = true, calls setTutorialData, calls progression.save().

`M.getCurrentStep()` -- Returns current_step (number). 0 means inactive.

`M.getStepCount()` -- Returns #STEPS[active_tutorial] or 0.

`M.getTutorialData()` -- Returns the tutorial_data from progression (convenience).

`M.setLang(language)` -- Sets lang to "en" or "ru".

**Persistence integration (lazy require to avoid circular deps):**
```lua
local function getProgression()
    return require("progression")
end
```

M.isDone reads: `getProgression().getTutorialData()[tutorial_id .. "_done"]`
M.markDone writes: updates the field, calls setTutorialData, calls save.

**EaseOutCubic (inline, matching codebase pattern):**
```lua
local function easeOutCubic(t)
    return 1 - (1 - t) * (1 - t) * (1 - t)
end
```

**Important: require("animation") is done lazily inside M.update() and M.advance() to avoid circular dependency issues. Do NOT put it at the module top level.**
