**mobile.lua changes:**
Add two new functions to the existing mobile module (append before `return mobile`):

1. `mobile.getPerformanceTier()` -- Returns cached string: `"HIGH"` (desktop), `"MED"` (web), `"LOW"` (native mobile). Detection logic:
   - If `mobile.isMobile()` returns true -> `"LOW"`
   - Else if `mobile.isWeb()` returns true -> `"MED"`
   - Else -> `"HIGH"`
   - Cache the result in a module-local `tier_cache` variable (set on first call, returned directly on subsequent calls).

2. `mobile.setPerformanceTier(tier)` -- Sets `tier_cache` to the given string. For future settings menu use. No validation needed (internal API).

Do NOT modify any existing functions. The existing `isLowPerformance()` function stays as-is for backward compatibility.

This module manages screen-level visual effects that are shared across both game modes: fly-to-bar resource icons, overlay flash, and celebration burst. It follows the same pre-allocated pool pattern as `particles.lua` for zero GC pressure during gameplay.

**Module structure:**
```lua
-- effects.lua
-- Screen-level visual effects: fly-to-bar icons, overlay flash, celebration burst
-- Pre-allocated pools for zero GC pressure during gameplay

local layout = require("layout")
local mobile = require("mobile")

local effects = {}
```

**1. Shared easing functions** (extract from existing code, centralize):
```lua
local function easeOutCubic(t)
    return 1 - (1 - t) * (1 - t) * (1 - t)
end

local function easeOutElastic(t)
    if t == 0 or t == 1 then return t end
    return math.pow(2, -10 * t) * math.sin((t * 10 - 0.75) * (2 * math.pi / 3)) + 1
end

local function easeInQuad(t)
    return t * t
end
```

**2. Resource bar target system** (for fly-to-bar -- targets set by screen modules):
```lua
local resource_targets = {
    fuel = {x = 0, y = 0},
    star = {x = 0, y = 0},
}

function effects.setResourceBarTargets(fuel_x, fuel_y, star_x, star_y)
    resource_targets.fuel.x = fuel_x
    resource_targets.fuel.y = fuel_y
    resource_targets.star.x = star_x
    resource_targets.star.y = star_y
end

function effects.getResourceTarget(icon_type)
    return resource_targets[icon_type]
end

-- Convenience: spawn fly icon toward stored resource target
function effects.spawnResourceFly(from_x, from_y, icon_type)
    local target = resource_targets[icon_type]
    if target and (target.x ~= 0 or target.y ~= 0) then
        effects.spawnFlyToBar(from_x, from_y, target.x, target.y, icon_type)
    end
end
```

**3. Fly-to-bar icon pool** (D-05: resource gain fly-to-bar):
- `MAX_FLY_ICONS = 15`
- Pre-allocate pool of icon objects: `{x, y, start_x, start_y, target_x, target_y, time, duration, icon_type, active, r, g, b}`
- Free-stack (same pattern as particles.lua)
- `effects.spawnFlyToBar(from_x, from_y, target_x, target_y, icon_type)` -- acquires from pool, sets fields, icon_type is `"fuel"` or `"star"`. Fuel color: `r=1, g=0.75, b=0.15`. Star color: `r=0.95, g=0.85, b=0.25`. Duration: tier-aware (0.6/0.5/0.4s).
- `effects.updateFlyIcons(dt)` -- update active icons: lerp position along arc (start -> target with arc height of 80px via `sin(pi * t)`), advance time, release when done.
- `effects.drawFlyIcons()` -- draw active icons as filled circles (radius 14 for fuel, 12 for star) with the icon color and alpha fading to 0.5 at end. Draw a white "+1" text centered above each icon using `love.graphics.print`.

**4. Overlay flash pool** (D-06: big reward flash):
- Module-local state: `flash_alpha = 0`, `flash_duration = 0`, `flash_time = 0`, `flash_r = 1, flash_g = 1, flash_b = 1`
- `effects.spawnFlash(duration, r, g, b)` -- sets flash state. Default color white (1,1,1). Default duration 0.3s. Alpha starts at 0.3.
- `effects.updateFlash(dt)` -- decrement `flash_time`, compute alpha as `0.3 * (1 - flash_time/flash_duration)` fading to 0.
- `effects.drawFlash()` -- if flash_alpha > 0, draw a fullscreen rectangle (`0, 0, layout.VW, layout.VH`) with `setColor(flash_r, flash_g, flash_b, flash_alpha)`.

**5. Celebration burst** (D-06: radial star burst for big moments):
- `MAX_BURST_PARTICLES = 20`
- Pre-allocate pool: `{x, y, vx, vy, time, duration, size, r, g, b, active}`
- `effects.spawnBurst(cx, cy, count, color)` -- spawns `count` (default 12, capped by tier: HIGH=20, MED=12, LOW=6) particles radiating outward from center point. Each gets random angle, speed 300-600, lifetime 0.5-0.8s, size 8-16.
- `effects.updateBurst(dt)` -- move particles, fade alpha, release when done.
- `effects.drawBurst()` -- draw as colored circles with alpha fade.

**6. Master update and draw:**
- `effects.init()` -- pre-allocate all pools. Read tier from `mobile.getPerformanceTier()` and store burst particle budget.
- `effects.update(dt)` -- calls `updateFlyIcons(dt)`, `updateFlash(dt)`, `updateBurst(dt)`.
- `effects.draw()` -- calls `drawFlyIcons()`, `drawBurst()`. Note: `drawFlash()` is separate because it needs to be drawn at a specific layer (above everything, below UI).
- `effects.drawFlash()` -- exposed separately for draw ordering control.

**7. Tier-aware budgets** stored in module local `tier_budgets`:
```lua
local TIER_BUDGETS = {
    HIGH = { burst_count = 20, fly_icon_duration = 0.6 },
    MED  = { burst_count = 12, fly_icon_duration = 0.5 },
    LOW  = { burst_count = 6,  fly_icon_duration = 0.4 },
}
```
