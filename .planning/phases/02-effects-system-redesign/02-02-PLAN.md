<objective>
Implement level-scaled merge celebration effects for Coin Sort mode, including coin fly-up on merge and fly-to-bar resource icons. Low merges (L2-L3) get a small pop, medium merges (L4) get standard celebration, and high merges (L5-L7) get full celebration with more particles, stronger shake, and brief flash. The merged result coin does a brief upward arc before settling. Resource gains (Fuel/Stars) spawn fly-to-bar icons from the merge point to the resource bar. This fulfills D-01 (satisfying pop style), D-02 (level scaling), D-03 (CS-specific chunky fragment aesthetic), D-05 (fly-to-bar in CS), and FX-03 (coin fly-up on merge).

Purpose: Make the merge loop feel progressively more rewarding as players achieve higher-level merges, which is core to the game's satisfying feedback loop. Fly-to-bar icons spatially connect merge rewards to the resource bar.
Output: Enhanced merge celebrations in Coin Sort with level-aware particle counts, shake intensity, flash effects, coin fly-up, and resource fly-to-bar.

**4. Add a celebration trigger function** (local, near the MERGE_CELEBRATION table):
```lua
-- Trigger level-scaled merge celebration (per D-01, D-02, D-03, FX-03)
local function triggerMergeCelebration(x, y, color, level)
    local c = MERGE_CELEBRATION[level] or MERGE_CELEBRATION[2]

    -- Extra particles scaled by level (on top of what animation.lua already spawns)
    if c.particle_mult > 1.0 then
        local extra = math.floor((c.particle_mult - 1.0) * 10)
        for i = 1, extra do
            particles.spawn(x, y, color)
        end
    end

    -- Enhanced screen shake (amplify the base shake from animation system)
    if c.shake_mult > 1.0 then
        local base_shake = animation.getShakeIntensity()
        animation.triggerShake(base_shake * c.shake_mult, 0.12)
    end

    -- Brief white flash for L4+ merges
    if c.flash > 0 then
        effects.spawnFlash(c.flash)
    end

    -- Coin fly-up: find a free fly-up slot and start arc animation
    for i = 1, MAX_FLY_UPS do
        if not fly_ups[i].active then
            fly_ups[i].active = true
            fly_ups[i].x = x
            fly_ups[i].y = y
            fly_ups[i].base_y = y
            fly_ups[i].time = 0
            fly_ups[i].r = color[1] or 1
            fly_ups[i].g = color[2] or 1
            fly_ups[i].b = color[3] or 1
            fly_ups[i].number = level
            break
        end
    end
end
```

**5. Add fly-up update and draw functions:**
```lua
local function updateFlyUps(dt)
    for i = 1, MAX_FLY_UPS do
        local fu = fly_ups[i]
        if fu.active then
            fu.time = fu.time + dt
            if fu.time >= FLY_UP_DURATION then
                fu.active = false
            else
                -- Parabolic arc: rises then falls back
                local t = fu.time / FLY_UP_DURATION
                local arc = math.sin(t * math.pi)  -- 0 -> 1 -> 0
                fu.y = fu.base_y - FLY_UP_HEIGHT * arc
            end
        end
    end
end

local function drawFlyUps()
    for i = 1, MAX_FLY_UPS do
        local fu = fly_ups[i]
        if fu.active then
            local t = fu.time / FLY_UP_DURATION
            local alpha = 1 - t * 0.5  -- fade slightly
            love.graphics.setColor(fu.r, fu.g, fu.b, alpha)
            -- Draw as a small tinted circle (like a coin)
            local r = layout.COIN_R * 0.6
            love.graphics.circle("fill", fu.x, fu.y, r)
            -- Number on the coin
            love.graphics.setColor(1, 1, 1, alpha)
            love.graphics.printf(tostring(fu.number), fu.x - 20, fu.y - 10, 40, "center")
        end
    end
end
```
