# Phase 1 -- UI Design Contract

> Visual and interaction contract for the Cleanup and Prep phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase scope note:** Phase 1 is primarily a backend/logic phase (dead code removal, save schema versioning, dead function removal). The single UI-visible deliverable is CLN-05: the double merge charge indicator on the Coin Sort screen. This contract specifies only the new UI element and documents existing visual tokens for consistency.

## Component Inventory -- New Elements (CLN-05 Only)

### 1. Double Merge Badge

**Location:** Top-right corner of the merge button (overlapping edge by 50%).
**Size:** 56px wide x 40px tall, rounded rectangle with 8px corner radius.
**Visual:**
- Background: `0.5, 0.85, 1.0` (solid light blue)
- Text: "2x" at 25px font size (font_small), white, centered
- Drop shadow: 2px offset, `0, 0, 0, 0.5` for readability over button edge

**Behavior:**
- Visible when `drops.getDoubleMergeCharges() > 0`
- Hidden (not drawn) when charges are 0
- Shows charge count if > 1: "2x (3)" where 3 is the charge count
- Badge text for single charge: "2x"
- Badge text for multiple charges: "2x x{N}" where N is charge count

**Draw order:** After `draw_merge_button()`, before `drawPowerupButtons()`. Badge draws on top of the merge button image.

### 2. Double Merge Consumption Popup

**Location:** Spawned at the merging box position (same as existing resource popups).
**Visual:** Uses the existing `spawnResourcePopup` pattern with custom text and color.

**Copy:** "DOUBLE MERGE!" (all caps for emphasis)
**Color:** `0.5, 0.85, 1.0` (matching the badge color for visual consistency)
**Animation:** Same float-up + fade-out as all existing resource popups (rise 80px over 1.2s, alpha 1->0).

### 3. Double Merge Charge Earned Notification (existing pattern, Arena side)

**Already exists:** `arena_screen.lua:1216` shows "+1 Double Merge!" via the arena notification system. No change needed.

## Interaction Contract

### Double Merge Badge Interaction

The badge is **display-only** -- it is not tappable. It communicates state passively.

| State | What Player Sees | What Happens |
|-------|-----------------|--------------|
| No charges | Merge button as normal (no badge) | Normal merge behavior |
| 1+ charges | "2x" badge on merge button corner | Next merge consumes 1 charge, doubles coin output |
| Charge consumed | Badge updates (decrements or disappears) + "DOUBLE MERGE!" popup | 4 output coins instead of 2 |

### Merge Button with Active Double Merge

When double merge charges > 0, the merge button gets a subtle pulsing glow effect:
- Glow color: `0.5, 0.85, 1.0, 0.3` (30% alpha light blue)
- Pulse frequency: 1.5 Hz (sin wave, alpha oscillates 0.15 to 0.30)
- Glow extends 4px beyond button bounds (drawn as a slightly larger rounded rect behind the button)
- Purpose: draws player attention to the special merge state without being distracting

## Implementation Notes for Executor

1. **Logic/visual separation (CLAUDE.md rule):** The charge check and consumption happen in `coin_sort.lua` (logic module). The badge drawing, glow effect, and popup spawning happen in `coin_sort_screen.lua` (screen module). Do NOT put drawing code in `coin_sort.lua` or `drops.lua`.

2. **Draw order in `coin_sort_screen.draw()`:** Insert double merge badge drawing after `draw_merge_button()` (line 813 of current code). The badge must layer on top of the button sprite.

3. **Existing popup system:** Use `spawnResourcePopup(from_x, from_y, {text = "DOUBLE MERGE!", color = {0.5, 0.85, 1.0}})` for the consumption notification. No new popup infrastructure needed.

4. **No new font objects needed:** The badge text uses the existing `font_small` (25px). No new LOVE2D font creation required.

5. **No new images needed:** The badge is drawn with `love.graphics.rectangle` + `love.graphics.printf`, matching the existing powerup button pattern.
