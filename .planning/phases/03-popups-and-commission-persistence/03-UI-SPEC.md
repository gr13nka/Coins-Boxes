# Phase 3 — UI Design Contract

> Visual and interaction contract for the Popups and Commission Persistence phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (LOVE2D immediate-mode UI) |
| Preset | not applicable |
| Component library | none (hand-rolled love.graphics primitives) |
| Icon library | none (text labels + geometric shapes) |
| Font | comic shanns.otf (bundled custom font) |

**Note:** This is a LOVE2D/Lua game, not a web frontend. All rendering is immediate-mode via `love.graphics` calls on a 1080x1920 virtual canvas. The shadcn gate is not applicable. Colors below are expressed as LOVE2D RGBA (0-1 floats).

---

## Spacing Scale

Declared values in virtual canvas pixels (1080x1920 coordinate space). All multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Cell gaps, inline padding between label and value |
| sm | 8px | Compact element spacing, panel internal padding |
| md | 16px | Corner radius for popup cards, default element margins |
| lg | 24px | Section padding within popup cards, toast internal padding |
| xl | 32px | Gap between popup card content sections |
| 2xl | 48px | Major spacing between popup title and content body |
| 3xl | 64px | Vertical margin above/below centered popup cards |

Exceptions:
- Toast banner height: 80px (matches tab bar height for visual consistency)
- Touch targets: minimum 80px height for all tappable buttons (matches existing `POWERUP_BTN_H` and `TAB_HEIGHT`)
- Commission row height: 80px minimum (up from current 36px, to accommodate progress bar + collect button)

## Typography

All fonts use "comic shanns.otf" loaded at different sizes. Sizes in virtual canvas units (1080-wide canvas).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 36px | regular (single weight font) | 1.4 | Commission descriptions, popup body text, toast messages |
| Label | 25px | regular | 1.3 | Difficulty badges, reward preview labels, progress fractions |
| Heading | 48px | regular | 1.2 | Popup titles ("Level Up!", "Order Complete!"), celebration card headlines |
| Display | 64px | regular | 1.1 | Level number in celebration popup only |

**Font loading:** Two new font sizes needed in `main.lua`:
- `font_heading = love.graphics.newFont("comic shanns.otf", 48)` -- popup titles
- `font_display = love.graphics.newFont("comic shanns.otf", 64)` -- celebration level number

Existing fonts reused:
- `font` (36px) = Body
- `font_small` (25px) = Label

**Note:** comic shanns.otf is a single-weight font. Bold/semibold is achieved by drawing text twice with 1px offset, or by using color/size contrast instead. Prefer size and color differentiation over faux-bold.

## Component Inventory

### 1. Toast Banner (D-01, D-05, D-10)

**Trigger events:** Commission completion, drop notifications (gen tokens, fuel surge, star burst, bag bundle), order completion rewards.

**Layout:**
```
+------------------------------------------------------+
| [4px accent stripe] [icon] [message text]       [x/2s]|
+------------------------------------------------------+
```

| Property | Value |
|----------|-------|
| Width | VW - 40 (1040px), centered horizontally |
| Height | 80px |
| Y position (resting) | 20px from top of canvas (below safe area) |
| Y position (hidden) | -100px (above canvas) |
| Corner radius | 12px |
| Background | (0.12, 0.16, 0.11, 0.92) |
| Border | (0.3, 0.5, 0.3, 0.5), 2px line width |
| Left accent stripe | 4px wide, accent green (0.25, 0.65, 0.35) |
| Text color | (0.92, 0.88, 0.78) at 36px body size |
| Icon area | 50x50px square, left-aligned after stripe, colored by resource type |

**Animation:**
- Slide in: Y from -100 to 20, duration 0.25s, ease-out cubic
- Hold: 2.0s visible (auto-dismiss)
- Slide out: Y from 20 to -100, duration 0.2s, ease-in quad
- Tap anywhere: immediate slide-out (0.15s)

**Stacking (Claude's Discretion):**
- Multiple toasts stack vertically: second toast at Y = 20 + 88 (80 + 8 gap), third at Y = 20 + 176
- Maximum 3 visible toasts. Additional toasts queue behind and appear as earlier ones dismiss.
- Each toast has independent dismiss timers.

### 4. Popup Queue System (D-08, D-09)

| Property | Value |
|----------|-------|
| Queue type | FIFO, global (survives screen switches) |
| Toast overlap | Yes, up to 3 visible simultaneously |
| Card/celebration overlap | No, one at a time, sequential |
| Inter-item delay | 0.3s between card/celebration dismiss and next popup enter |
| Input blocking | Cards and celebrations block all game input. Toasts do not block input. |
| Timer behavior | Game timers (free bag, generator recharge) keep ticking during popups (D-08) |

**Priority ordering:** If both toasts and a card are queued, the card displays first (higher priority). Toasts can display simultaneously alongside cards only if the toast was already visible before the card appeared.

## Interaction Contract

### Popup Input Blocking (D-08, D-11)

```lua
-- Input priority chain in every screen's mousepressed():
-- 1. Card/celebration popup (blocks all input, requires accept button tap)
-- 2. Toast banner (does NOT block input, tap-anywhere dismisses the toast)
-- 3. Normal game input
```

**Card/celebration dismiss:** Only the accept button dismisses. Tapping outside the card does nothing (prevents accidental dismiss of important rewards).

**Toast dismiss:** Tapping anywhere on the toast banner dismisses it. Tapping elsewhere in the game proceeds normally (toasts are non-blocking).

### Commission Collect Flow (D-14)

1. Commission completes -> entry gets pulsing green border + "Collect" button replaces progress area
2. Player taps "Collect" -> rewards applied immediately (bags.addBags, resources.addStars)
3. Reward fly-to-bar animation plays
4. Toast fires: "Commission Complete! +{N} Bags +{N} Stars"
5. Entry shows collected state (checkmark, dimmed)
6. When both commissions collected -> 0.3s pause -> batch refresh generates 2 new commissions (D-13)
7. New commissions fade in

### Popup Queue Flow (D-09)

1. Event fires (level up, commission complete, drop notification)
2. `popups.push({tier, title, body, rewards, onDismiss})` adds to FIFO queue
3. Queue processor checks: if no card/celebration currently showing, show next
4. For toasts: show immediately (up to 3 stacked)
5. For cards/celebrations: wait for any existing card/celebration to be dismissed + 0.3s delay
6. On dismiss: `onDismiss` callback fires, queue advances

## Drawing Order (Z-Order)

Strict layer order in every screen's `draw()` function. This extends the Pattern 2 from ARCHITECTURE.md.

```
1. Background + game content (grid, coins, boxes)
2. Particles (particles.draw())
3. Effects (effects.draw() -- fly-to-bar, burst, flash)
4. HUD (resource bars, commission panel)
5. Toast banners (popups.drawToasts())
6. Card/celebration popup with dimmer (popups.drawModal())
7. Tab bar (tab_bar.draw())
```

**Key decision:** Tab bar draws ABOVE popups. This ensures screen switching is always accessible, even during reward popups. This is consistent with D-08 (no gameplay pause -- player should be able to navigate).

**Exception:** During celebration popups (level up), tab bar is still visible but input is blocked by the popup layer. The visual hierarchy makes the popup feel like a momentary overlay, not a screen takeover.
