Create `popups.lua` as a new data+rendering module following the project's `local popups = {} ... return popups` pattern. This module handles all three popup tiers and the queue system.

**Module structure:**

1. **Constants (from UI-SPEC):**
   - Toast: width=1040, height=80, Y_VISIBLE=20, Y_HIDDEN=-100, corner_radius=12, bg=(0.12,0.16,0.11,0.92), border=(0.3,0.5,0.3,0.5), accent_stripe=(0.25,0.65,0.35), text_color=(0.92,0.88,0.78), MAX_VISIBLE_TOASTS=3, HOLD_TIME=2.0, SLIDE_IN_TIME=0.25, SLIDE_OUT_TIME=0.2, TAP_DISMISS_TIME=0.15
   - Medium card: width=648, min_height=360, max_height=600, corner_radius=16, bg=(0.12,0.16,0.11,0.95), border=(0.3,0.5,0.3,0.6), border_width=3, backdrop=(0,0,0,0.7), button_w=300, button_h=70, button_radius=12, button_color=(0.25,0.65,0.35)
   - Celebration card: width=864, min_height=480, max_height=720, corner_radius=20, bg=(0.12,0.16,0.11,0.95), border=(0.35,0.75,0.45,0.8), border_width=4, title_color=(0.95,0.85,0.25), button_w=350, button_h=80
   - Text primary: (0.92,0.88,0.78), text secondary: (0.65,0.68,0.58)
   - Queue inter-item delay: 0.3s

2. **State:**
   - `queue` -- FIFO array of `{tier="toast"|"card"|"celebration", title=string, body=string, rewards={{icon_type, amount},...}, onDismiss=function|nil}`
   - `active_toasts` -- array (max 3) of `{data, state="entering"|"visible"|"exiting", timer, y, alpha, scale}`
   - `active_modal` -- nil or `{data, state="entering"|"visible"|"exiting", timer, alpha, scale, backdrop_alpha}`
   - `inter_delay_timer` -- countdown between sequential cards/celebrations
   - `fonts` -- {heading, display, body, label} set via init()

3. **Easing functions** (local, matching effects.lua pattern):
   - `easeOutCubic(t)` = `1 - (1-t)^3`
   - `easeInQuad(t)` = `t*t`
   - `easeOutElastic(t)` -- for celebration bounce (same formula as effects.lua)

4. **Public API:**

   `popups.init(fonts_table)` -- receives `{heading=font_heading, display=font_display, body=font, label=font_small}`. Stores in module-local `fonts`. Initializes empty queue and state.

   `popups.push(item)` -- item is `{tier, title, body, rewards, onDismiss}`. Appends to queue FIFO. For tier="toast": if fewer than MAX_VISIBLE_TOASTS active, immediately spawn. For tier="card"|"celebration": queue processes in update.

   `popups.update(dt)` -- Updates all active toasts (timers, state transitions, removal). Updates active_modal (enter/exit animations). Processes queue: if no active_modal and inter_delay_timer <= 0, pop next card/celebration from queue and start entering. Toasts are popped from queue independently (up to MAX_VISIBLE_TOASTS active). The queue is priority-sorted: cards/celebrations before toasts when deciding what to process next (per UI-SPEC priority ordering).

   `popups.drawToasts()` -- Draws all active toasts. Each toast: rounded rect bg with 2px border, 4px accent stripe on left edge (accent green), icon area 50x50 left-aligned after stripe colored by first reward's icon_type (or white if text-only), message text at 36px body font in text_primary. Y position per toast: Y_VISIBLE + i*(80+8). Animation: entering = lerp Y from Y_HIDDEN to target, exiting = lerp Y from target to Y_HIDDEN.

   `popups.drawModal()` -- If active_modal exists: draw full-canvas backdrop (0,0,0, backdrop_alpha). Draw card: for "card" tier use medium card dimensions, for "celebration" use celebration dimensions. Card content: title at heading font centered, body at body font centered below title with 48px gap, reward row (horizontal icons + amounts) centered, accept button at bottom. Button text: "Claim Reward" for card, "Continue" for celebration. Celebration extras: on enter, call `effects.spawnFlash(0.3, 0.95, 0.85, 0.25)` and `effects.spawnBurst(VW/2, VH_VISIBLE/2, burst_count, {0.95, 0.85, 0.25})` where VH_VISIBLE is approx 960 (vertical center of visible area). Scale animation: entering cards scale 0.85->1.0 (easeOutCubic, 0.2s), celebration scales 0.7->1.0 (easeOutElastic, 0.3s). Exiting: scale 1.0->0.9 + alpha fade (0.15s card, 0.2s celebration).

   `popups.isModalActive()` -- Returns true if active_modal is not nil (any state including exiting).

   `popups.isInputBlocked()` -- Returns true if active_modal is in "entering" or "visible" state (blocks game input per D-08, D-11).

   `popups.handleToastTap(x, y)` -- Check if (x,y) hits any active visible toast. If so, start that toast exiting (TAP_DISMISS_TIME). Returns true if a toast was dismissed. Called by screen modules before game input per D-10.

   `popups.handleModalTap(x, y)` -- If active_modal is in "visible" state, check if (x,y) hits the accept button rect. If so, start exit animation, fire onDismiss callback. Returns true if tapped. Only the button dismisses per D-11 -- tapping outside does nothing.

5. **Drawing reward icons:** For each reward in `rewards` array, draw a small colored circle (fuel=gold, star=yellow, bag=brown using ICON_COLORS table matching effects.lua pattern) with "+" and amount text beside it. Use font_small (25px). Icons laid out horizontally, centered in card.

6. **Logic/visual note:** This module contains BOTH logic (queue management) and rendering because popup rendering is inherently tied to popup state (animation timers, positions). This follows the same pattern as `tab_bar.lua` which also combines state+rendering for a UI overlay component. The key pattern is maintained: popups.lua does NOT touch game state (no resources, bags, coins). It only manages its own queue and visual state. The `onDismiss` callback pattern lets callers handle game-state side effects.

**Do NOT use goto.** Use simple if/elseif chains for state transitions.
